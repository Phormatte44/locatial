# Locatial Canonical Truths

These are the highest-authority product truths.

1. Library is what the creator has.
2. Projects are what the creator makes.
3. Published is what the audience sees.
4. A Profile may own multiple Channels.
5. A Channel may publish multiple Projects.
6. Projects should eventually publish under a Channel identity, not only a user identity.
7. A Place Node is created once and referenced many times.
8. Place truth and project meaning are separate.
9. A Story Stop may reference a Place Node, but it does not require one.
10. A located Stop may include camera framing: zoom, pitch, bearing.
11. Datasets live in Library.
12. Dataset features are not automatically Places.
13. Dataset features can become visual layers, Place Nodes, both, or neither.
14. Datasets require source URL and attribution.
15. The map is a spatial lens, not the editor.
16. Preview is part of authoring, not just final QA.
17. Agent drafts are internal/admin seeding tooling for Locatial, not a creator-facing feature yet.
18. Do not create separate editors for Story, Guide, and Collection.
19. Do not build a full GIS, social network, marketplace, or freeform page builder until explicitly adopted.
20. First prototype uses React, Tailwind CSS, shadcn/ui, and Obra shadcn_ui kit Pro edition 1.13.2 in neutral dark mode.
