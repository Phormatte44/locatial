# Skill — Visual Prototype

Use this for early dummy-data prototypes.

## Goal

Create a rich visual prototype that helps Michael feel the product before backend integration.

## Optimize for

- Premium interaction clarity.
- Clear three-column composition.
- Dummy data that feels real.
- Clickable interactions over completeness.
- Fast iteration.
- No backend dependency.
- No authentication.
- No production schema enforcement.

## Must include in first prototype

- Story Stops left rail.
- Stop Editor center.
- Research Tray right rail.
- Dummy global Library.
- Stop-specific Context Notes.
- Spatial Context drawer.
- Basic Preview mode.
- Dummy Channel identity.
- Dummy Place assignment states.
- Camera fields: zoom, pitch, bearing.

## Do not

- Connect Supabase yet.
- Build production auth.
- Build full dataset import.
- Build full GeoJSON editor.
- Over-engineer component architecture.
- Hide behind generic placeholder UI.
