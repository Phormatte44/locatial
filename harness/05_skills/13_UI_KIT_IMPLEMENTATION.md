# Skill — UI Kit Implementation

Use this when building the first visual prototype and early Creator Studio UI.

## Required UI stack for prototype

- React
- Tailwind CSS
- shadcn/ui
- Obra shadcn_ui kit — Pro edition 1.13.2

## Intent

This first pass is not a final visual-design exploration.

The goal is to test interaction, structure, layout, and product model using a neutral, reliable component system.

## Visual direction for now

- Dark mode only.
- Neutral shadcn-style interface.
- Generic but polished.
- No custom brand-heavy styling.
- No experimental visual language.
- No new design system invention.
- Use standard component patterns first.

## Use Obra / shadcn components for

- Buttons
- Cards
- Inputs
- Textareas
- Tabs
- Dialogs
- Drawers / Sheets
- Dropdown menus
- Badges
- Scroll areas
- Separators
- Tooltips
- Command palette if needed

## Do not

- Invent bespoke components unless required.
- Pull visual references from previous Locatial mockups.
- Use neon/globe/Space Odyssey styling.
- Over-style the prototype.
- Make visual design the evaluation target.
- Use non-shadcn component libraries.
- Build with plain HTML if a React/Tailwind/shadcn app is being initialized.
