# Skill — Database / Supabase

Use this only when moving from prototype to persistence.

## Principles

- Schema follows product model.
- Place truth and project meaning stay separate.
- Dataset and Layer are separate.
- Draft/publish must be explicit.
- Channel relationship should be represented early.

## Do not

- Start backend integration before prototype flow is approved.
- Store secrets in client markup.
- Use public write RLS beyond POC.
