# Skill — Frontend Implementation

Use this for React/HTML/CSS/JS implementation.

## Prototype phase

- Prioritize one working route.
- Use dummy data.
- Keep state simple and inspectable.
- Prefer clear components.
- Avoid premature libraries.
- Make the UI feel good enough to judge interactions.

## Later production phase

- Add typed data contracts.
- Move dummy data to fixtures.
- Connect Supabase only after the interaction model is validated.

## Do not

- Build backend before the prototype proves the flow.
- Hide interaction behind non-working buttons.
- Create brittle one-off CSS everywhere.
