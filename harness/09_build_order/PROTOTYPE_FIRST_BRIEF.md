# Prototype First Brief

## Objective

Build a richer visual prototype before building production infrastructure.

The first prototype should help Michael evaluate the core authoring model.

## Required implementation stack

Use:

- React
- Tailwind CSS
- shadcn/ui
- Obra shadcn_ui kit — Pro edition 1.13.2

## Visual constraint

Use a neutral dark-mode shadcn-style interface.

Do not use previous Locatial visual references for this first pass. The goal is interaction validation, not final art direction.

## Required screen

Creator Studio desktop screen with:

1. Left column: Story Stops
2. Middle column: Stop Editor
3. Right column: Research Tray

## Required dummy story

Use a realistic dummy project:

**Perfect Friday Night in Oakland**

Stops:

- Opening
- Prescott Market
- Walk between stops
- Swan Market
- Final recommendation

## Required interactions

- Select Stop.
- Add Stop.
- Rename Stop.
- Show Stop status.
- Show assigned / candidate / unassigned place.
- Add text block.
- Add quote block.
- Add image placeholder.
- Move Library item into Stop Context.
- Move Stop Context note into Stop Editor.
- Mark note as Used.
- Open Spatial Context drawer.
- Assign dummy place.
- Adjust dummy camera fields: zoom, pitch, bearing.
- Open Preview mode.
- Preview stop-by-stop story.

## Do not build

- Supabase.
- Auth.
- Real dataset import.
- Full map library.
- Production publishing.
- AI agent UI.
- Custom Locatial visual design language.
